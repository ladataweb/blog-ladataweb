La popularidad de Power Bi es cada vez mayor y muchos developers en el mundo quiere aportar, a la industria de Bi, herramientas que ayuden la práctica diaria. Siendo el software de Microsoft el principal monstruo del mercado, la tendencia es fortalecer tareas junto a éste.

Hace más de un mes que Power Bi Desktop a anunciado una característica muy interesante que abre la puerta a muchos desarrollos independientes que puede unirse y acompañar el desarrollo de proyectos de la mejor forma posible. Tal vez ya conocíamos algunas de las herramientas principales que se conectaban a un modelo tabular. Éstas ahora pueden integrarse en Desktop para facilitar la conexión y el uso.

External Tools es, por el momento, solo puede usarse mediante un preview feature dentro de Power Bi Desktop que se activan en el menú de Opciones bajo características preliminares. Es necesario que desktop considere a su modelo tabular de cierta característica para permitir el acceso externo al mismo. Técnicamente esta liberando el acceso a los metadatos del modelo que poseerán un estilo semejante a Analysis Services.

La opción que vamos a activar es: Uso de metadatos de conjunto de datos mejorado.

Luego de reiniciar tras activar la opción, nuestro Power Bi Desktop permitirá la conexión a los metadatos del conjunto de datos de otras herramientas. Para ver el menú inicial es necesario tener una de las tres principales herramientas asociadas a Power Bi:

Si tenemos instalada la ultima versión de alguna de las tres pintadas, el menú aparecerá para poder utilizarlo. Una breve descripción de ellas:

  • Tabular Editor permite a los profesionales de BI compilar, mantener y administrar fácilmente modelos tabulares mediante un editor intuitivo y ligero. Una vista jerárquica muestra todos los objetos del modelo tabular, organizados por carpetas para mostrar con compatibilidad para la edición de propiedades de selección múltiple y el resaltado de sintaxis de DAX. El código fuente de Tabular Editor se puede encontrar en el siguiente repositorio de GitHub: Tabular Editor en GitHub
  • DAX Studio es conocido por ser una completa herramienta para la creación, el diagnóstico, el ajuste del rendimiento y el análisis de DAX. Entre sus características se incluyen la exploración de objetos, el seguimiento integrado, los desgloses de la ejecución de consultas con estadísticas detalladas y resaltado y aplicación de formato de sintaxis de DAX. En la siguiente imagen se muestra una pantalla de DAX Studio. El código fuente de DAX Studio se puede encontrar en el siguiente repositorio de GitHub: DAX Studio en GitHub
  • ALM Toolkit es una herramienta de comparación de esquemas para conjuntos de datos de Power BI, usada para escenarios de administración del ciclo de vida de las aplicaciones (ALM). Con ella, puede realizar una implementación sencilla en todos los entornos y retener los datos históricos de actualización incremental. Con ALM Toolkit, puede comparar y combinar archivos de metadatos, ramas y repositorios. También puede reutilizar definiciones comunes entre conjuntos de datos. El código fuente de ALM Toolkit se puede encontrar en el siguiente repositorio de GitHub: ALM Toolkit en GitHub

Éstas son las más populares y las que darán aparición a nuestro menú de "External Tools". Cabe aclarar que no son las únicas. La comunidad ha aprovechado este lanzamiento para construir cada vez más herramientas que ayuden a nuestro desarrollo. Algunos ejemplos de ellos son document model, open in excel, open in tableau, etc. 

Seguramente pronto veremos mucho más que nos ayude a mejorar nuestros procesos y proyectos de Bi gracias a las nuevas herramientas de soporte que vayan saliendo.
