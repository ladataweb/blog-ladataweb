Excelente llegada luego de 4 largos meses para Power Bi Report Server. El servidor de reportes on premise se pone al día con todas las increíbles features de Power Bi que se estuvieron lanzando en los últimos meses.

Los detalles sobre los cambios son los siguientes:

Reporting

  • Cross-highlight on a single point in line charts
  • Word wrap on titles
  • Update default visual interaction to cross-filter
  • Rounded corners for visual borders
  • Single select slicer
  • Heat map support for Bing maps
  • Cross-highlight by axis labels
  • Default tooltip formatting
  • Static web URL support for buttons, shapes & images
  • Page alignment options
  • Selection pane improvements
  • Accessible visual interaction
  • Conditional formatting for visual titles 
  • Conditional formatting for web URL actions for buttons, shapes, and images
  • Performance analyzer pane
  • Table and matrix keyboard navigation
  • Line data label position control
  • KPI visual Indicator text size control

Analytics

  • Show dates as a hierarchy now generally available

Modeling

  • New modelling view generally available
  • New DAX functions
  • Update to the ALLSELECTED DAX function
  • Disable auto-date tables for new reports

El cambio estructural más grande es New modelling view generally available. La nueva vista de relaciones que permite interactuar cambios a múltiples tablas, columnas e incluso crear carpetas entre ellas. Ésta nos permite crear pequeñas vistas de tablas involucradas que nos podrían servir para documentar mejor nuestro desarrollo.

Cabe destacar tres particulares que me hacen dar ganas de modificar mis reportes bajo un nuevo plano que estuve esperando tanto tiempo. Las mismas son:

Rounded corners for visual borders: Muchos podrán pensar que es un cambio estético e innecesario, pero es cierto que tantos cuadrados por todas partes quitan un toque estético importante que Power Bi debió tener hace tiempo, pues muchos la eligen por su gran facilidad para armar una buena estética.

Single select slicer: amé este cambio. Hace tiempo que vengo hablando con el soporte de Power Bi para explicarles que tocar "Ctrl" en un slicer y filtrar más de uno es un bug y no algo funcional puesto que ya existe la selección multiple con checks. No quisieron hacerme caso pero finalmente ahora hay raddio buttons para casos en que realmente debe ser una sola opción la que hay que seleccionar.

Heat map support for Bing maps: si mal no recuerdo la necesidad de esta idea la vote en el año 2016. Es uno de los mapas más solicitados por entidades varias y tenerlo nativamente en el mapa de bing es una gran fortaleza para no estar obligado a usar ArcGis (que funciona lento en power bi) o la "custom visual Heatmap" que requiere pequeños motores gráficos.
